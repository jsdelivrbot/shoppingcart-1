Parent components aren't destroyed when navigating to children so state is automatically maintained. When you click a breadcrumb to come back here, this textbox value remains, even if it isn't bound to anything.

This component has a value sticky = 1 so that it still displays when its child route is active.

Child textbox value: "{{ childTextboxValue }}". This was received via (fromChild).

It navigates to and sends its child data via [toChild] when an item is clicked. When subsequent items are clicked, the route is not reloaded (since it doesn't change) but the child receives the data again as an event.
